# Appium 自动化脚本录制器 - 使用指南

## 一、快速开始

### 1. 前置条件

- Node.js 已安装
- 手机已开启 USB 调试并连接电脑
- **ANDROID_HOME 环境变量已配置**

### 2. 一键配置

首次使用时，技能会自动：
1. 检查环境依赖
2. 引导安装缺失的依赖
3. 自动创建 `driver.js` 基础模块
4. 生成配置文件模板

### 3. 配置 App 信息

编辑文件 `appium-tests/config/appium.config.js`：

```javascript
module.exports = {
  server: {
    hostname: 'localhost',
    port: 4723
  },
  android: {
    platformName: 'Android',
    'appium:automationName': 'UiAutomator2',
    'appium:deviceName': 'Android',

    // ⚠️ 必须修改：App 包名和启动 Activity
    'appium:appPackage': '你的App包名',
    'appium:appActivity': '你的启动Activity',

    'appium:noReset': true,
  },
  accounts: {
    default: {
      account: '你的测试账号',
      password: '你的测试密码'
    }
  }
};
```

### 4. 如何获取 App 包名和 Activity

**方法1：通过 ADB 查询（推荐）**

```bash
# 1. 在手机上打开目标 App
# 2. 执行以下命令查询当前运行的 App
adb shell dumpsys window windows | grep mCurrentFocus
```

## 二、使用方式

### 方式1：智能录制（推荐）

**触发词：** `录制脚本`、`录制 xxx 页面的脚本`

**示例：**
```
录制卖出页面的脚本
```

**录制流程：**
1. 技能自动检查环境依赖
2. 用户描述操作（如 "点击卖出"）
3. 技能智能匹配候选元素
4. 用户快速确认（只看 1-3 个候选）
5. 完成后自动生成脚本

### 方式2：交互示例

```
[步骤 1] 请描述下一步操作: 点击卖出

智能匹配结果：
  [1] 卖出按钮 ✓已验证
      定位器: //*[@resource-id="btn_sell"]
      稳定性: ⭐⭐⭐ 高
      位置: 屏幕右下角

  [2] 卖出 ✓已验证
      定位器: //*[@content-desc="卖出"]
      稳定性: ⭐⭐ 中
      位置: 屏幕底部

请选择: 1

✓ 已记录目标元素: //*[@resource-id="btn_sell"]
✓ Fallback 坐标: (720, 960)

请在手机上完成操作，完成后按回车继续...
```

## 四、智能匹配原理

### 用户只需描述操作

```
用户输入: "点击卖出"
         ↓
技能自动:
  1. 提取关键词: 动作="点击"，目标="卖出"
  2. 搜索匹配元素
  3. 按稳定性排序
  4. 验证定位器有效性
  5. 显示位置信息
         ↓
输出: 1-3 个候选，用户快速确认
```

### 稳定性评分

| 定位器类型 | 稳定性 | 说明 |
|-----------|--------|------|
| resource-id | ⭐⭐⭐ 高 | App 更新后通常保持稳定 |
| content-desc | ⭐⭐ 中 | 无障碍描述，UI 变化可能影响 |
| text | ⭐ 低 | 文案变化或多语言会失效 |
| 坐标定位 | ⭐ 低 | 不同设备可能失败 |

## 五、PopupWindow 兼容

### 自动检测

技能会自动检测 PopupWindow 场景：
- 元素数量突变 > 10
- 采集成功率 < 50%

### 增强策略

| 参数 | 普通页面 | PopupWindow |
|------|----------|-------------|
| 超时时间 | 3秒 | 8秒 |
| 重试次数 | 3次 | 5次 |

### Fallback 机制

生成的脚本包含双重定位策略：

```javascript
async clickConfirm() {
  try {
    // 主定位器（优先）
    const element = await driverFactory.waitForElement('//*[@resource-id="confirm"]');
    await element.click();
  } catch (e) {
    // Fallback: 坐标定位（兜底）
    await driverFactory.tapCoordinate(540, 960);
  }
}
```

## 六、常见问题

### Q1: Appium 服务未运行

**错误信息：** `Appium 服务未运行 (127.0.0.1:4723)`

**解决方法：**
```bash
# 安装 Appium
npm install -g appium
appium driver install uiautomator2

# 启动服务
appium
```

### Q2: 设备未连接

**错误信息：** `未检测到已连接的设备`

**解决方法：**
1. 检查 USB 线连接
2. 开启手机 USB 调试
3. 执行 `adb devices` 确认设备已识别

### Q3: ANDROID_HOME 未配置

**解决方法：**
```bash
# Windows
setx ANDROID_HOME "你的Android SDK路径"

# macOS/Linux
export ANDROID_HOME=你的Android SDK路径
```

### Q4: 智能匹配不到元素

**解决方法：**
- 选择 `[m] 更多匹配结果` 查看更多候选
- 选择 `[c] 手动输入定位器` 自定义 XPath
- 选择 `[s] 使用坐标定位` 使用坐标点击
