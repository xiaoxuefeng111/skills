# Appium 自动化流程审核与脚本生成器

## 版本说明

**v3.0 智能学习版**：
- **记忆学习**：记住用户偏好，下次优先推荐常用定位器
- **动态评分**：根据历史成功率动态调整推荐排序
- **烟花效果**：录制成功后显示欢快烟花动画
- **录制交互**：明确的选项列表（继续/完成/回放/摘要）
- **智能匹配**：用户描述操作 → 自动匹配候选元素 → 快速确认
- **定位器验证**：推荐前验证定位器有效性
- **Fallback 坐标**：主定位器失败时使用坐标定位
- **PopupWindow 增强**：智能等待 + 多次重试

## 工作方式

本技能采用智能引导式录制：

```
用户描述操作 → 智能匹配候选 → 用户快速确认 → 记录步骤 → 生成脚本
```

## ⚠️ 前置条件

1. Appium 服务运行在 `127.0.0.1:4723`
2. 设备已连接并开启 USB 调试
3. 目标 App 已打开
4. 已配置 `appium.config.js`（首次使用会引导）

## ⚠️ 核心交互原则

### 所有用户交互必须支持手动输入

**原则：智能推荐只是辅助，用户始终可以自定义输入。**

执行本技能时，**每次调用 AskUserQuestion 都必须满足以下要求：**

1. **必须有输入入口**：用户可以通过 "Other" 选项输入自定义内容
2. **提示语要明确**：问题文案必须包含 "或输入自定义描述" 的提示
3. **推荐选项上限 3-4 个**：不要列出过多选项，保持简洁
4. **不单独设"手动输入"选项**：手动输入是每个交互的默认能力，不是额外选项

**正确的交互示例：**

```
问题：请选择目标元素，或输入自定义描述（如：点击持仓列表第一行）

选项：
  [1] 卖出 ⭐⭐⭐ 高稳定性
  [2] 买入 ⭐⭐ 中稳定性
  [3] 撤单 ⭐⭐ 中稳定性

Other: [用户可在此输入自定义描述]
```

### 静默执行原则

**依赖检查和配置创建应该静默完成，不输出中间步骤。**

| 要求 | 说明 |
|------|------|
| 不输出诊断信息 | 不要输出"Node.js 不在 PATH"、"正在安装依赖"等中间信息 |
| 自动修复优先 | 缺失的依赖、配置文件应该自动创建，不询问用户 |
| 只报告关键问题 | 只有无法自动修复的问题才提示用户 |
| 简洁状态报告 | 环境检查只输出一行结果："✓ 环境就绪" 或 "✗ 缺少xxx" |

**错误示例（不要输出这些）：**

```
❌ Node.js 不在 PATH 中，让我找到 Node.js 的位置
❌ 缺少 webdriverio 模块，需要安装依赖
❌ 需要配置文件。让我先检查并创建配置
❌ 配置文件不存在，需要获取当前 App 信息
```

**正确做法：**

```
✓ 静默查找 Node.js 路径并使用
✓ 静默安装缺失的依赖
✓ 静默创建配置文件
✓ 静默获取 App 信息

环境检查完成后只输出：
"✓ 环境就绪，开始录制"
```

**错误的交互示例：**

```
❌ 错误：把手动输入做成单独选项
问题：请选择目标元素

选项：
  [1] 卖出
  [2] 买入
  [3] 撤单
  [4] 手动输入 ← 这是错误设计
```

## 录制流程

### 首先选择生成模式

技能触发后，首先询问用户选择生成模式：

```
请选择脚本生成模式：

  [1] 手动生成 (Recommended) - 交互式录制，逐步确认每个操作
  [2] 自动生成 - 提供步骤列表，一键生成脚本

Other: [可输入自定义描述]
```

**默认选择手动生成**，用户可切换为自动生成。

### 模式一：自动生成

**适用场景**：用户清楚知道要执行的所有步骤

**流程**：
1. 用户输入步骤列表（自然语言描述）
2. 技能自动解析步骤
3. 逐步执行并采集元素
4. 生成完整脚本

**用户输入示例**：

```
1. 启动应用 com.tdx.AndroidtdxOem 入口activity是com.tdx.Android.TdxAndroidActivity
2. 切换到交易 Tab
3. 点击卖出按钮
4. 输入账号 010000035265
5. 输入密码 123321
6. 点击最下面的交易登录，背景是红色的
7. 界面上会有输入股票4个字，点击一下
8. 输入股票代码 508033
9. 输入卖出数量 100
10. 点击卖出按钮
11. 点击确认卖出按钮
12. 点击我知道了
13. 完成
```

**执行过程**：
```
✓ 环境就绪，开始自动生成

[步骤 1/13] 启动应用...
  ✓ 应用已启动

[步骤 2/13] 切换到交易 Tab...
  智能匹配: "交易" ⭐⭐⭐ 高稳定性
  ✓ 已点击

[步骤 3/13] 点击卖出按钮...
  智能匹配: "卖出" ⭐⭐ 中稳定性
  ✓ 已点击

...（中间步骤省略）...

[步骤 13/13] 完成
  ✓ 脚本已生成: appium-tests/test-sell-flow.js
```

**智能匹配策略**：
- 每个步骤自动匹配最佳元素
- 优先使用高稳定性定位器
- 匹配失败时自动使用坐标 fallback
- 无需用户确认，全程自动

### 模式二：手动生成

**适用场景**：需要精确控制每个操作，或步骤不确定时

**流程**：
1. 环境检查（静默完成）
2. 确认页面信息
3. 智能录制（逐步确认）
4. 生成脚本

### 步骤 1: 环境检查（静默）

技能自动检查并修复：
- Node.js 环境
- Appium 服务
- 设备连接
- 依赖模块
- 配置文件

只输出最终结果："✓ 环境就绪" 或 "✗ 无法修复: xxx"

### 步骤 2: 确认页面信息

询问用户：
- 页面名称（如: position, buy, sell）
- 当前是否在正确的界面

### 步骤 3: 智能录制

```
[步骤 1] 请描述下一步操作: 点击卖出

智能匹配结果：
  [1] 卖出按钮 ✓已验证
      定位器: //*[@resource-id="btn_sell"]
      稳定性: ⭐⭐⭐ 高
      位置: 屏幕右下角

  [2] 卖出 ✓已验证
      定位器: //*[@content-desc="卖出"]
      稳定性: ⭐⭐ 中
      位置: 屏幕底部

  [m] 更多匹配结果
  [c] 手动输入定位器
  [s] 使用坐标定位

请选择: 1
```

### 步骤 4: 执行操作

用户在手机上执行操作，技能采集验证结果。

### 步骤 5: 录制结束交互

每次步骤完成后，显示明确的选项菜单：

```
步骤 X 已完成！请选择下一步：

  [1] 继续录制 - 添加更多操作步骤
  [2] 完成录制 - 生成测试脚本
  [3] 回放验证 - 运行刚才录制的步骤
  [4] 查看摘要 - 显示已录制的所有步骤

Other: [输入自定义操作]
```

### 步骤 6: 录制成功反馈

录制成功后显示烟花效果：

```
    🎉🎉🎉 录制成功！🎉🎉🎉

        ✨ *  ✨  *  ✨
      *    🎊    *    🎊
        ✨ *  ✨  *  ✨
           🎯 完美！

    已记录您的选择，下次会更聪明哦~
```

### 步骤 7: 生成脚本

自动生成：
- Page Object 文件（含 fallback 坐标）
- 测试脚本（可直接运行）

## 学习机制（v3.0 新增）

### 核心原理

技能会记住用户的选择历史，下次遇到相同元素时优先推荐用户偏好的定位器。

```
第一次录制：用户选择 resource-id 定位器 → 记录偏好
第二次录制：优先推荐 resource-id（标注"用户偏好"）
录制的越多 → 推荐越精准
```

### 数据存储

学习数据存储在 `rules/${appPackage}.json`，按 App 隔离。

```json
{
  "appPackage": "com.tdx.AndroidtdxOem",
  "lastUpdated": "2026-04-03T22:00:00",
  "totalSessions": 15,
  "elementKnowledge": {
    "卖出": {
      "preferredLocator": "resource-id",
      "locators": {
        "resource-id": { "value": "btn_sell", "usageCount": 12, "successCount": 11 }
      }
    }
  }
}
```

### 智能推荐效果

**使用前（静态评分）**：
```
智能匹配结果：
  [1] 卖出 ⭐⭐⭐ 高稳定性
  [2] 卖出 ⭐⭐ 中稳定性
```

**使用后（动态评分 + 历史偏好）**：
```
智能匹配结果：
  [1] 卖出 ✓已验证 ✓用户偏好
      定位器: //*[@resource-id='btn_sell']
      稳定性: ⭐⭐⭐ 高 | 历史成功率: 91.7% (11/12次)
      💡 您过去12次录制中，11次选择了此定位器

  [2] 卖出 ✓已验证
      定位器: //*[@content-desc='卖出']
      稳定性: ⭐⭐ 中 | 历史成功率: 100% (3/3次)
```

如果多个 App 属于同一系列（如不同券商的通达信），可合并学习数据：


## 智能匹配原理

### 匹配流程

```
用户描述 "点击卖出"
       ↓
关键词提取：动作="点击"，目标="卖出"
       ↓
元素搜索：text/content-desc/hint 包含 "卖出"
       ↓
稳定性排序：resource-id > content-desc > text
       ↓
定位器验证：实际尝试定位元素
       ↓
输出推荐：只展示 top 3 候选
```

### 稳定性评分

| 定位器类型 | 稳定性 | 评分 |
|-----------|--------|------|
| resource-id | ⭐⭐⭐ 高 | 10/10 |
| content-desc | ⭐⭐ 中 | 7/10 |
| hint | ⭐⭐ 中 | 6/10 |
| text | ⭐ 低 | 4/10 |
| 坐标定位 | ⭐ 低 | 3/10 |

## PopupWindow 兼容

### 检测机制

- 元素数量突变 > 10
- 采集成功率 < 50%

### 增强策略

| 参数 | 普通页面 | PopupWindow |
|------|----------|-------------|
| 超时时间 | 3秒 | 8秒 |
| 重试次数 | 3次 | 5次 |
| 稳定阈值 | 3次 | 5次 |

### Fallback 机制

生成的脚本包含双重定位策略：

```javascript
async clickConfirm() {
  try {
    // 主定位器
    const element = await driverFactory.waitForElement('//*[@resource-id="confirm"]', 5000);
    await element.click();
  } catch (e) {
    // Fallback: 坐标定位
    await driverFactory.tapCoordinate(540, 960);
  }
}
```
